ML DDoS Shield

A machine-learning powered DDoS detection and mitigation system that uses behavioral analytics, anomaly detection, a game-based CAPTCHA, and an IP honeypot trap to neutralize malicious traffic — without disrupting real users.

6
Game CAPTCHA — Dino Run
Hour 11–15
▶

When a session scores suspicious, a Dino-run game is served. The player must reach score 150 to receive a session token. HMAC signing prevents bots from faking the score.

Middleware triggers challengesuspicious score detected
↓
GET /captcha/new_gameserver issues game_id + HMAC fragment
↓
Canvas game servedSpace / click / touch to jump
↓
Player scoresobstacles spawn, speed increases
↓
Score ≥ 150?
↓
PassHMAC verified → session token issued
Failfail count + 1, retry
↓
3 fails → block IPRedis flag + honeypot
! The HMAC secret_fragment is generated server-side per game session. A bot cannot POST a fake score of 999 — the server will reject it because the signature won't match.

06 · api

API Reference

POST
/api/score
Score a session's behavioral features. Returns action: allow / captcha / redirect + anomaly score.
GET
/api/dashboard
Returns last 100 events, flagged IPs list, and 30-minute traffic sparkline from Redis.
POST
/api/captcha/pass
Mark IP as verified human after CAPTCHA pass. Whitelists in Redis for 1 hour.
POST
/api/captcha/fail
Record CAPTCHA failure. Flags IP on 3rd failure and returns status: flagged.
GET
/captcha/challenge
Serve the Dino-run game CAPTCHA HTML page.
POST
/captcha/verify
Verify game score + HMAC signature. Issues session token on pass.
GET
/captcha/new_game
Issue a new game_id and HMAC secret_fragment for a fresh game session.
07 · features

Behavioral Feature Table

Every session is scored on these 8 signals computed over a 10-second rolling window per IP.

FeatureHuman rangeBot signatureWhy it matters
click_count3–2050–300+Bots click far more per window
avg_click_interval400–2000ms<100msInhuman speed is the clearest tell
click_interval_variance50k–350k<200Humans have natural timing jitter
click_interval_entropy2.0–4.0 bits<0.5Bot timing patterns have near-zero entropy
mouse_velocity_variance0.2–1.5~0No mouse movement = no browser / mobile tap
max_element_click_rate0.1–4.0/s10–60/sHammering one endpoint is a DDoS signature
scroll_events2–150 or 1000+Real pages get scrolled; raw HTTP calls don't
keystroke_count5–800 or uniformZero keys = no interaction; uniform = paste bot
08 · demo

Demo Script — For Judges

Run this during the presentation. It fires a realistic bot signature at the live ngrok URL and shows the detection working in real time.

simulate_bot_demo.py
# Paste your ngrok URL below then run: python simulate_bot_demo.py
 
import requests, time
 
BASE = "https://YOUR-NGROK-URL.ngrok-free.app"
 
bot_payload = {
  "session_id": "demo-bot-001",
  "click_count": 200,
  "avg_click_interval": 30,
  "click_interval_variance": 8,
  "click_interval_entropy": 0.02,
  "mouse_velocity_variance": 0,
  "max_element_click_rate": 40,
  "scroll_events": 0,
  "keystroke_count": 0,
}
 
for i in range(10):
  r = requests.post(f"{BASE}/api/score", json=bot_payload)
  print(r.json())
  time.sleep(0.5)
→ Expected output: {"action": "redirect", "score": 0.95, "label": "bot"} — watch the detect.py terminal turn red simultaneously.
Talking point 1

Most DDoS protection breaks the user experience. Our game CAPTCHA is fun — real users barely notice and often want to keep playing.

Talking point 2

We compute Shannon entropy of click intervals and mouse velocity variance — signals that are nearly impossible for bots to fake without a full browser simulation.

Talking point 3

Confirmed bots are not blocked — they are trapped. The attacker believes they are still hitting the real system while every request they send is logged and their bandwidth is wasted on tar-pit delays.
